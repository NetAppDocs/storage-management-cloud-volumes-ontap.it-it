È possibile distribuire Cloud Volumes ONTAP su Amazon Web Services (AWS) e Microsoft Azure `Quick create` per una distribuzione più rapida sia in configurazioni a nodo singolo che ad alta disponibilità (HA). Questo processo semplificato riduce i passaggi di distribuzione rispetto al metodo avanzato, impostando automaticamente i valori predefiniti in un'unica pagina e minimizzando la navigazione.

.Dettagli del sistema
. *Account/credenziali del cloud provider*: i dettagli dell'account e dell'abbonamento vengono compilati automaticamente in base al tuo Console agent. Se disponi di più account o abbonamenti, seleziona quello che desideri utilizzare. Se un Console agent non è disponibile, ti verrà richiesto di https://docs.netapp.com/us-en/console-setup-admin/concept-agent-install-options.html["creare un agente Console"^].
. *Nome*: Immettere un nome per il sistema Cloud Volumes ONTAP (chiamato anche nome del cluster). La Console utilizza il nome del sistema (cluster) per denominare il sistema Cloud Volumes ONTAP e l'istanza di macchina virtuale. Utilizza inoltre il nome come prefisso per il gruppo di sicurezza predefinito, se si seleziona tale opzione.
. *Credenziali ONTAP*: Queste sono le credenziali per l'account amministratore del cluster Cloud Volumes ONTAP. Puoi utilizzare queste credenziali per connetterti a Cloud Volumes ONTAP tramite ONTAP System Manager o la ONTAP CLI. Puoi mantenere il nome utente predefinito _admin_ o cambiarlo con un nome utente personalizzato.
. *Tag*: I tag sono i metadati per le risorse AWS o Azure. La Console aggiunge i tag all'istanza Cloud Volumes ONTAP e a ciascuna risorsa associata all'istanza. È possibile aggiungere fino a 15 tag dall'interfaccia utente durante la creazione di un sistema Cloud Volumes ONTAP e successivamente aggiungerne altri dopo la creazione. Per ulteriori informazioni, consultare:

.Distribuzione e configurazione
. *Tipo/topologia di implementazione*: Seleziona il tipo di implementazione che desideri utilizzare: nodo singolo, alta disponibilità (HA) in una singola zona di disponibilità (AZ) o HA in più AZ.
. *Configurazione di rete*: Inserisci le informazioni di rete che hai registrato nel foglio di pianificazione per regioni, VNets e sottoreti.
+
.. *Regione*: Per impostazione predefinita, viene selezionata la regione dell'account cloud associato alle risorse.
.. *Zona di disponibilità*: È possibile selezionare la zona di disponibilità in base al tipo di implementazione scelto e se la regione selezionata supporta la zona di disponibilità.
.. *Sottorete*: Selezionare una sottorete per la regione.
+

.Disponibilità elevata di AWS
Se hai selezionato una configurazione HA, inserisci le seguenti informazioni:

.HA in AZ singola
... *Accesso mediatore*: specificare le informazioni di accesso del mediatore.  Il mediatore è un'istanza separata che monitora lo stato di salute della coppia HA e fornisce il quorum in caso di errore.  Fornire il nome della coppia di chiavi per consentire all'istanza del mediatore di connettersi al servizio AWS EC2 e selezionare il metodo di connessione.


.HA in più AZ
... *Zone di disponibilità e mediatore*: seleziona le zone di disponibilità (AZ) per ciascun nodo e il mediatore e le subnet corrispondenti in cui desideri distribuire la coppia Cloud Volumes ONTAP HA.
... *IP mobili*: se si scelgono più AZ, specificare gli indirizzi IP mobili per i servizi NFS e CIFS e per la gestione di cluster e SVM.  Gli indirizzi IP devono essere esterni al blocco CIDR per tutte le VPC nella regione.  Per ulteriori dettagli, fare riferimento alink:https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/reference-networking-aws.html#requirements-for-ha-pairs-in-multiple-azs["Requisiti di rete AWS per Cloud Volumes ONTAP HA in più AZ"^] .
... *Accesso mediatore*: specificare le informazioni di accesso del mediatore.  Il mediatore è un'istanza separata che monitora lo stato di salute della coppia HA e fornisce il quorum in caso di errore.  Fornire il nome della coppia di chiavi per consentire all'istanza del mediatore di connettersi al servizio AWS EC2 e selezionare il metodo di connessione.
... *Tabelle di routing*: se hai scelto più AZ, seleziona le tabelle di routing che includono i percorsi verso gli indirizzi IP mobili.  Se si dispone di più di una tabella di routing, è importante selezionare le tabelle di routing corrette.  In caso contrario, alcuni client potrebbero non avere accesso alla coppia Cloud Volumes ONTAP HA.  Per ulteriori informazioni sulle tabelle di routing, fare riferimento a http://docs.aws.amazon.com/AmazonVPC/latest/UserGuide/VPC_Route_Tables.html["Documentazione AWS: tabelle di routing"^] .

.Ricarica e servizi

Per informazioni sulle diverse licenze, consulta link:concept-licensing.html["Scopri di più sulle licenze Cloud Volumes ONTAP"^].
. *Servizi e funzionalità dati*: mantieni abilitati i servizi o disabilita quelli che non desideri utilizzare con Cloud Volumes ONTAP.
+
** https://docs.netapp.com/us-en/data-services-data-classification/concept-classification.html["Scopri di più sulla classificazione NetApp"^]
** https://docs.netapp.com/us-en/data-services-backup-recovery/concept-backup-to-cloud.html["Scopri di più su NetApp Backup and Recovery"^]

+

TIP: Se si desidera utilizzare WORM e il tiering dei dati, è necessario disabilitare Backup e ripristino e distribuire un sistema Cloud Volumes ONTAP con versione 9.8 o successiva.



. * Account del sito di supporto NetApp *: se disponi di più account, seleziona quello che desideri utilizzare.
. Controlla o modifica i dettagli inseriti, quindi fai clic su *Crea*.


ifdef::azure[]

In Azure, la sezione *Panoramica* mostra le risorse che verranno allocate al termine del processo di distribuzione.

endif::azure[]

ifdef::aws[]

In _AWS_, subito dopo aver creato il sistema Cloud Volumes ONTAP, la NetApp Console avvia un'istanza di test nella VPC specificata per verificare la connettività. In caso di esito positivo, la Console termina immediatamente l'istanza e quindi avvia la distribuzione del sistema. Se la Console non riesce a verificare la connettività, la creazione del sistema fallisce. L'istanza di test può essere una  `t2.nano` (per tenancy VPC predefinita) o una  `m3.medium` (per tenancy VPC dedicata).

endif::aws[]

.Dopo aver finito
La console distribuisce il sistema Cloud Volumes ONTAP .  È possibile monitorare i progressi nella pagina Audit.

Se riscontri problemi durante la distribuzione del sistema Cloud Volumes ONTAP, esamina il messaggio di errore. Puoi anche selezionare *Ricrea ambiente* sul sistema per continuare con la distribuzione utilizzando la modalità  `Advanced` procedura guidata.


CAUTION: Una volta completata la procedura di distribuzione, non modificare le configurazioni di Cloud Volumes ONTAP generate dal sistema nel portale cloud di AWS o Azure, in particolare i tag di sistema. Qualsiasi modifica apportata a queste configurazioni potrebbe causare comportamenti imprevisti o la perdita di dati.
